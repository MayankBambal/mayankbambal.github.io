Project Overview

A comprehensive Python-based tool for automatically extracting and processing structured data from unstructured documents using OpenAI's GPT models. While the repository demonstrates extraction from SEC 10-K filings, the underlying framework is designed to be general-purpose and can be adapted for various document types and data extraction needs.

View on GitHub

Key Features

  • Automated Document Download: Downloads documents from various sources (demonstrated with SEC EDGAR database)
  • AI-Powered Data Extraction: Uses OpenAI GPT models for intelligent data extraction from unstructured text
  • Dual Extraction Methods: Basic and enhanced extraction with confidence tracking and probability scoring
  • Advanced Data Cleaning: Preprocesses HTML/CSS and converts tables to markdown for optimal AI processing
  • Comprehensive Logging: Detailed logging and monitoring of extraction processes with error handling
  • Structured Output: Exports data to CSV with metadata, confidence scores, and validation mechanisms
  • Customizable Templates: Adaptable extraction templates for different document types and use cases

Technical Architecture

Core Components

  • Downloader Module: Fetches documents and organizes them into structured directories
  • Preprocessor: Cleans HTML content and converts tables to markdown format
  • Basic Extractor: Core extraction engine using OpenAI's API
  • Enhanced Extractor: Advanced extraction with confidence assessment and probability tracking
  • Table Converters: HTML and plain text to markdown conversion utilities

Processing Pipeline

  1. Download: Fetches documents from target sources
  2. Clean: Removes formatting and converts tables to structured markdown
  3. Extract: Uses OpenAI GPT models to extract structured data
  4. Process: Structures data into pandas DataFrames with validation
  5. Export: Saves results to CSV with comprehensive metadata

Technologies Used

  • Python 3.8+ - Core programming language
  • OpenAI GPT Models - AI-powered text understanding and extraction
  • Pandas - Data manipulation and analysis
  • BeautifulSoup/HTML Parsing - Document preprocessing
  • Logging Framework - Comprehensive monitoring and debugging
  • CSV Export - Structured data output format
  • Environment Management - Secure API key handling

Project Structure

src/
├── main.py                    # Basic extraction pipeline
├── main_enhanced.py           # Enhanced pipeline with probability tracking
├── downloader.py              # Document downloader (adaptable to various sources)
├── preprocessor.py            # HTML cleaning and preprocessing
├── extractor.py               # Basic OpenAI-powered extraction
├── enhanced_extractor.py      # Enhanced extraction with confidence metrics
├── html_to_table.py           # HTML table to markdown converter
├── plain_to_table.py          # Plain text table to markdown converter

data/
├── raw/                       # Downloaded documents
├── processed/                 # Cleaned text files
├── final/                     # Basic extraction results
└── enhanced_final/            # Enhanced extraction results

Output & Results

The system generates structured CSV files with comprehensive metadata including:

  • Basic Output: Extracted data with item descriptions, values, units, and source metadata
  • Enhanced Output: Includes AI confidence scores and classification (HIGH/MEDIUM/LOW confidence)
  • Validation: Error handling and data quality assessment
  • Traceability: Complete audit trail from source document to extracted data

Applications

While demonstrated with SEC financial filings, this general-purpose framework can be adapted for:

  • Financial document processing and analysis
  • Legal document data extraction
  • Research paper data mining
  • Healthcare record processing
  • Government document analysis
  • Any structured data extraction from unstructured text sources

Note: The GitHub repository demonstrates SEC financial data extraction as a reference implementation, but the underlying codebase is designed to be general-purpose and can be customized for various document types and extraction requirements.
